## 四、建议执行顺序

```
第 1 周前半：阶段 1（Dify 集成）+ 阶段 2（权限初始化）
            → 里程碑：用户可通过 Dify 界面提问，权限隔离生效

            → 里程碑：检索质量达标，管理员可操作文档生命周期

第 2 周前半：阶段 4（自动过期）+ 阶段 8（测试加固）+ 阶段 9（文档培训）
            → 里程碑：系统可上线试运行

            → 里程碑：高级功能可用
```

## 五、风险与阻塞项

| 风险 | 影响 | 应对 |
|------|------|------|
| Dify 版本不兼容 | Chatflow HTTP 节点功能受限 | 确认 Dify 版本 ≥ 1.13 |
| 知识库目录权限 | 用户无法写入共享文件夹 | 需要 IT 配置 SMB/NFS ACL |
| 模型效果不理想 | 回答质量差、用户不信任 | 阶段 5 重点优化，准备 benchmark |
| GPU 显存不足 | 多模型并发时 OOM | 当前 16GB 够用，监控使用量 |
| 用户抵触 | 不愿用新系统 | 阶段 9 培训 + 渐进推广 |

**已完成：** P0 基础搭建的核心部分（后端 + 入库 + 检索 + 部署）

**下一步：** 优先完成阶段 1（Dify 集成），让系统从"开发者能用"变成"普通用户能用"。
